Frequently Asked Questions

1. What is the difference between `var`, `let`, and `const`?
var is function-scoped, allows re-declaration, and is hoisted with undefined. let and const are block-scoped, with let allowing reassignment but not re-declaration, while const prohibits both (except object mutations). Both are hoisted but uninitialized, raising a ReferenceError if accessed before declaration. Use let or const for better scoping and clarity.
2. What is the difference between `map()`, `forEach()`, and `filter()`?
map() transforms an array and returns a new array with modified elements. forEach() iterates over an array but doesn’t return a new one. filter() returns a new array containing elements that meet a specific condition. Use map() for transformation, forEach() for side effects, and filter() for filtering data.
3. Explain arrow functions and how they are different from regular functions.
Arrow functions are a more concise way to write functions in JavaScript using =>. Unlike regular functions, they don’t have their own this context but inherit it from their surrounding scope, making them ideal for callbacks. They cannot be used as constructors and don’t have a prototype. Use arrow functions for shorter syntax and when you don’t need a separate this.
4. How JavaScript Promises work?
JavaScript Promises handle asynchronous operations by representing a value that might be available now, later, or never. They have three states: `pending`, `fulfilled`, and `rejected`. You use `.then()` for success, `.catch()` for errors, and `.finally()` for cleanup, enabling cleaner asynchronous code compared to callbacks. Promises improve readability and avoid callback nesting.
5. How closures work in JavaScript?
Closures in JavaScript occur when a function "remembers" the variables from its outer scope even after the outer function has finished executing. This allows inner functions to access and manipulate those variables. Closures are useful for creating private variables, maintaining state, and building powerful abstractions like function factories or event handlers.
